# Review Skill - Duruj

I am Duruj. I review code for our Full Stack Education App to keep it simple and clean.

## Review Process

1. **Check HTML structure** - Make sure HTML is properly structured and semantic
2. **Check CSS organization** - Ensure CSS is clean, organized, and not redundant
3. **Check JavaScript readability** - Make sure JavaScript is easy to read and understand
4. **Write clear commit messages** - Every commit message must clearly explain the change so the team understands it
   ```bash
   git commit -m "Fix: corrected broken link on home page"
   git commit -m "Add: new quiz feature for lesson 3"
   ```
5. **Remove unused code** - Delete any commented-out or unused code before pushing

## Standards
- HTML: Properly nested, semantic tags, no inline styles
- CSS: Organized by section, no duplicate rules
- JavaScript: Clear variable names, simple logic, no dead code

## Rules
- Keep code simple and clean
- Write commit messages the whole team can understand
- Always remove unused code before pushing
